# WireGuard Split Tunneling - Раздельное туннелирование

## 📋 Описание проекта

Проект предназначен для настройки **раздельного туннелирования (split tunneling)** в WireGuard на Windows. Это позволяет направлять трафик определенных доменов через локальную сеть (в обход VPN), в то время как остальной трафик идет через VPN-туннель.

### Зачем это нужно?

- **Российские сервисы** (Госуслуги, банки, госучреждения) часто блокируют доступ через VPN
- **Локальные ресурсы** могут быть недоступны при активном VPN-подключении
- **Оптимизация скорости** - локальный трафик не проходит через удаленные серверы
- **Избежание блокировок** - некоторые сайты определяют и блокируют VPN-трафик

## 🚀 Быстрый старт

### Шаг 1: Настройка конфигурации

Откройте файл `domains.conf` и добавьте домены, которые должны работать **в обход VPN**:

```conf
# Один домен на строку!
# Комментарии начинаются с #

# Госуслуги
gosuslugi.ru
esia.gosuslugi.ru
gu-st.ru

# Банки
sberbank.ru
tinkoff.ru

# Другие сервисы
ozon.ru
vk.com
```

### Шаг 2: Изменение шлюза по умолчанию

В файлах `add_routes.ps1` и `remove_routes.ps1` измените переменную `$Gateway` на IP-адрес вашего **локального шлюза**:

```powershell
$Gateway = "192.168.1.1"  # Замените на ваш шлюз
```

**Как узнать свой шлюз:**
1. Откройте командную строку (cmd)
2. Выполните команду: `ipconfig`
3. Найдите строку **"Основной шлюз"** в вашем активном подключении

### Шаг 3: Запуск скрипта добавления маршрутов

**ВАЖНО:** Запускайте PowerShell **от имени администратора**!

#### Обычный запуск (маршруты удаляются после перезагрузки):

```powershell
# Правой кнопкой мыши на PowerShell → "Запуск от имени администратора"
cd C:\Users\fuji2\Documents\VPN\WireguardClientExApp\Wireguard-split-tunneling
.\add_routes.ps1
```

#### С постоянными маршрутами (сохраняются после перезагрузки):

```powershell
.\add_routes.ps1 -p
```

**Параметр `-p`:** делает маршруты постоянными - они сохраняются после перезагрузки системы.

**Когда использовать `-p`:**
- Если не хотите настраивать автозагрузку скрипта
- Для тестирования - чтобы маршруты оставались после перезагрузки

**Важно:** Скрипт автоматически удаляет старые маршруты перед добавлением новых (даже с `-p`). 

**⚠️ КРИТИЧЕСКИ ВАЖНО при изменении `domains.conf`:**
Если вы **удаляете домены** из списка, нужно сделать так:
1. Запустите `remove_routes.ps1` (**ДО** изменения файла!)
2. Измените `domains.conf` (удалите ненужные домены)
3. Запустите `add_routes.ps1` (или `add_routes.ps1 -p`)

Если забыть шаг 1, маршруты для удаленных доменов останутся в таблице маршрутизации. Их можно удалить вручную командой:
```powershell
route delete <IP-адрес>
```
Или посмотреть все маршруты и удалить ненужные:
```powershell
route print
```

Скрипт автоматически:
- Определит активный сетевой интерфейс
- Прочитает домены из `domains.conf`
- Разрешит каждый домен в IP-адрес
- Добавит статические маршруты через локальный шлюз

### Шаг 4: Проверка работы

После выполнения скрипта проверьте маршруты:

```powershell
route print | findstr "192.168.3.1"
```

Или протестируйте доступность сайтов при активном VPN.

## 📖 Подробное описание скриптов

### Общий конфигурационный файл: domains.conf

**Оба скрипта используют один и тот же файл `domains.conf`!**

Этот файл содержит список доменов, для которых нужно управлять маршрутами.

**Формат файла:**
- Один домен на строку
- Комментарии начинаются с `#`
- Пустые строки игнорируются
- Поддомены нужно указывать отдельно (например, `api.example.com` и `example.com`)

**Пример содержимого:** см. раздел "Шаг 1: Настройка конфигурации" выше

---

### add_routes.ps1

**Назначение:** Добавляет статические маршруты для доменов из `domains.conf` через локальный шлюз.

**Как работает:**
1. Читает список доменов из `domains.conf`
2. Для каждого домена:
   - Разрешает доменное имя в IP-адреса (IPv4)
   - **Удаляет существующий маршрут** (если есть) - строка 29
   - Добавляет новый маршрут с метрикой 1 через локальный шлюз
3. Выводит результат операции

**Важная особенность:** Скрипт автоматически удаляет старые маршруты перед добавлением новых (строка 29: `route.exe delete $ipStr`). Поэтому не нужно запускать `remove_routes.ps1` перед `add_routes.ps1`.

**Параметры:**
- `$Gateway` - IP-адрес локального шлюза (по умолчанию: 192.168.3.1)
- `$Mask` - маска подсети (по умолчанию: 255.255.255.255)
- `$IfIndex` - определяется автоматически
- **`-p`** - опция командной строки для создания постоянных маршрутов (сохраняются после перезагрузки)

**Когда использовать:**
- После подключения к WireGuard VPN
- При изменении списка доменов в `domains.conf`
- После перезагрузки системы (если настроена автозагрузка)

**Пример вывода:**
```
Добавлен маршрут: 185.165.123.45 -> 192.168.3.1 (metric 1, if 12)
Обработано: gosuslugi.ru IP: 185.165.123.45
```

---

### remove_routes.ps1

**Назначение:** Удаляет все маршруты для доменов из `domains.conf`.

**Как работает:**
1. Читает список доменов из `domains.conf`
2. Для каждого домена разрешает имя в IP-адреса
3. Удаляет все найденные маршруты
4. Подтверждает выполнение операции

**Когда использовать:**
- При необходимости полностью очистить все маршруты (без добавления новых)
- Перед отключением VPN
- Для troubleshooting - проверить, что проблема именно в маршрутах

**Важно:** Оба скрипта работают с одним файлом `domains.conf`. Изменения в этом файле влияют на работу обоих скриптов.

## 🔧 Возможные проблемы и их решение

### Проблема: "Отказано в доступе"

**Причина:** Скрипт запущен без прав администратора.

**Решение:**
1. Закройте текущее окно PowerShell
2. Нажмите `Win + X`
3. Выберите "Windows PowerShell (администратор)"
4. Запустите скрипт снова

### Проблема: "Не удалось найти IP для domain.ru"

**Причины:**
- Домен не существует или опечатка
- Проблемы с DNS
- Домен временно недоступен

**Решения:**
1. Проверьте правильность написания домена
2. Попробуйте выполнить `nslookup domain.ru` в командной строке
3. Проверьте настройки DNS в свойствах сетевого подключения:
   - Нажмите `Win + R`, введите `ncpa.cpl`
   - Правой кнопкой на активном подключении → Свойства
   - Выберите "IP версии 4 (TCP/IPv4)" → Свойства
   - Укажите альтернативные DNS: `8.8.8.8` и `1.1.1.1`

### Проблема: Маршруты не работают после добавления

**Причины:**
- Неверный IP шлюза
- Конфликт с другими маршрутами
- WireGuard перезаписывает маршруты

**Решения:**
1. Проверьте правильность `$Gateway` в скриптах
2. Выполните `route print` и проверьте наличие маршрутов
3. Запускайте `add_routes.ps1` **после** подключения к WireGuard
4. **ВАЖНО:** В настройках туннеля WireGuard **ОТКЛЮЧИТЕ** опцию "Block untunneled traffic (kill-switch)" - WireGuard по умолчанию ставит наивысший приоритет своим маршрутам, и эта опция блокирует локальные маршруты

### Проблема: После перезагрузки маршруты исчезли

**Причина:** Статические маршруты, добавленные через `route.exe`, не сохраняются после перезагрузки.

**Решение:** См. раздел "Автоматический запуск при старте системы" выше

## ⚙️ Настройка WireGuard клиента

### Отключение Kill-Switch (ОБЯЗАТЕЛЬНО!)

**ВАЖНО:** Для работы split tunneling необходимо отключить опцию kill-switch в WireGuard!

1. Откройте клиент WireGuard
2. Нажмите кнопку **Edit** рядом с вашим туннелем
3. **СНИМИТЕ галочку** с опции **"Block untunneled traffic (kill-switch)"**
4. Нажмите **Save**

![Kill-switch настройка](image/kill-switch.png)

**Почему это важно:**
- WireGuard по умолчанию блокирует весь трафик, который не идет через VPN
- Это предотвращает работу локальных маршрутов, которые вы добавляете через скрипты
- Без отключения этой опции ваши исключения доменов НЕ БУДУТ работать

### Порядок действий:

1. **Отключите kill-switch** в настройках туннеля WireGuard
2. Подключитесь к VPN
3. Запустите `add_routes.ps1` от имени администратора
4. Проверьте работу исключенных доменов

##  Автоматический запуск при старте системы

Статические маршруты не сохраняются после перезагрузки. Есть два способа решить это:

### Способ 1: Использовать параметр -p (простой)

Запустите скрипт с параметром `-p` один раз - маршруты сохранятся навсегда:

```powershell
.\add_routes.ps1 -p
```

**Плюсы:**
- Не нужно настраивать автозагрузку
- Маршруты сохраняются после перезагрузки

**Минусы:**
- При удалении доменов из `domains.conf` нужно: сначала `remove_routes.ps1`, потом изменить файл, потом `add_routes.ps1 -p`
- Если IP домена изменится, маршрут останется старым до следующего запуска скрипта

---

### Способ 2: Автозагрузка скрипта (гибкий)

1. Нажмите `Win + R`, введите `shell:startup`
2. Создайте ярлык для `add_routes.ps1` в открывшейся папке
3. В свойствах ярлыка → Дополнительно → Запуск от имени администратора

**Плюсы:**
- Маршруты всегда актуальные (IP разрешаются заново при каждом запуске)
- Легко обновлять список доменов

**Минусы:**
- Требует настройки автозагрузки

---

### Способ 3: Планировщик заданий (для продвинутых)

```powershell
# Выполнить от имени администратора
$action = New-ScheduledTaskAction -Execute "PowerShell.exe" -Argument "-ExecutionPolicy Bypass -File `"C:\Users\fuji2\Documents\VPN\WireguardClientExApp\Wireguard-split-tunneling\add_routes.ps1`""
$trigger = New-ScheduledTaskTrigger -AtLogOn
$principal = New-ScheduledTaskPrincipal -UserId "SYSTEM" -LogonType ServiceAccount -RunLevel Highest
Register-ScheduledTask -TaskName "WireGuard Routes" -Action $action -Trigger $trigger -Principal $principal
```

## ⚠️ Важные предупреждения

1. **Безопасность:** Трафик, идущий в обход VPN, не шифруется и виден вашему провайдеру
2. **DNS-утечки:** Убедитесь, что DNS-запросы также идут через нужный интерфейс
3. **Конфликты:** Не добавляйте слишком много маршрутов - это может замедлить работу сети
4. **Обновление IP:** Если сайт меняет IP-адрес, нужно перезапустить `add_routes.ps1`
5. **IPv6:** Скрипты работают только с IPv4. Для IPv6 нужна отдельная настройка

## 🔄 Обновление списка доменов

### Если вы ДОБАВЛЯЕТЕ домены:
Просто добавьте их в `domains.conf` и запустите:
```powershell
.\add_routes.ps1
```
Скрипт сам удалит старые маршруты и добавит новые.

### Если вы УДАЛЯЕТЕ домены:
**⚠️ ВАЖНО:** Сначала очистите маршруты, ПОТОМ изменяйте файл!

```powershell
# 1. Запустите удаление (читает текущий список)
.\remove_routes.ps1

# 2. Теперь измените domains.conf (удалите ненужные домены)

# 3. Добавьте актуальные маршруты
.\add_routes.ps1
```

**Почему такой порядок?** `remove_routes.ps1` читает домены из `domains.conf`. Если вы сначала удалите домены из файла, скрипт не узнает какие маршруты нужно удалить!

## 🛠️ Расширенная настройка

### Изменение метрики маршрута

Метрика определяет приоритет маршрута. Чем меньше число, тем выше приоритет.

В `add_routes.ps1` измените параметр `metric`:
```powershell
route.exe add $ipStr mask $Mask $Gateway metric 1 if $IfIndex
#                                          ^^^^^^^^
#                                          Измените это значение
```

### Добавление целых подсетей

Если нужно направить всю подсеть через локальный шлюз, добавьте вручную:
```powershell
route.exe add 192.168.100.0 mask 255.255.255.0 192.168.3.1 metric 1 if 12
```

### Проверка DNS

Убедитесь, что DNS-запросы идут через правильный интерфейс:
```powershell
ipconfig /all
# Проверьте "DNS-серверы" для каждого адаптера
```
